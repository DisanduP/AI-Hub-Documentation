Overview

AI Hub is a centralized management and observability platform for AI agents at 99x. It enables developers to register agents, monitor performance in real-time, and manage organizational access through a unified dashboard.

Architecture

Designed as a high-performance system with clear layer separation:

  • Frontend: React/Next.js dashboard for management and visualization.
  • Backend: ASP.NET Core API Gateway handling business logic and external integrations.
  • Integration: Seamless connection with Azure OpenAI, Microsoft Graph, and Entra ID (Azure AD).
  • Persistence: PostgreSQL for relational data and potentially vector stores for agent memory.

Core Data Model

Entities & Relationships

  • Users: Managed via Office 365/Entra ID. Supports multi-tenancy.
  • Agents: Each agent instance belongs to exactly one team.
  • Teams: Supports hierarchical structures (Parent/Child teams).
  • Roles: Access is managed at the agent/team level (Admin vs. User).
Entity Key Attributes
AiAgent Category, Status, AI Model, TeamId
AgentTeam Name, Description, ParentTeamId
AgentMetric Name, Value, Unit, Dimensions (JSON)

Tech Stack & Infrastructure

Cloud Native: Hosted entirely on Azure using Infrastructure as Code principles.

  • API: ASP.NET Core 8.0 Web API with Entity Framework Core.
  • Database: Azure Database for PostgreSQL.
  • Auth: Microsoft Identity Web (Entra ID) with JWT and custom tenant validation.
  • Observability: Application Insights telemetry integrated into both FE and BE.
Component Technology
Frontend App Next.js (Azure Static Web Apps)
Backend API .NET Core (Azure App Service)

Secrets Management Azure Key Vault

Scope & Features

Agent Management

Full CRUD for agents you belong to, including category tagging (Status, Model, Category) and contact person assignment.

Team Governance

Manage members, roles (Admin/User), and team hierarchies. Invite members and handle access requests.

Identity

RBAC aligned to org hierarchy with external identity support (AWS IAM, Azure AD).

API & Security

Secure communication is established via Agent-specific API keys.

  • Key Management: Handled at the Agent/Team level.
  • Auth: OIDC and API Keys for metrics ingestion.
  • Endpoints: CRUD for entities, Async push for logs/metrics, and Real-time triggers.

Getting Started: Quick Start Guide

1. Register Your AI Agent

Register your agent to get a unique identifier and enable monitoring.

curl -X POST http://localhost:5000/agents \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{
    "name": "Customer Support Chatbot",
    "description": "AI-powered customer support assistant",
    "category": "CustomerService",
    "model": "GPT4",
    "keyFeatures": "NLP;Intent classification"
  }'

2. Create API Key for Metrics

Generate a key for your agent to send metrics without full user authentication.

curl -X POST http://localhost:5000/agents/{agentId}/apikeys \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{"name": "Metrics Key"}'

3. Send Metrics

Use your API key to push performance data.

curl -X POST http://localhost:5000/agents/{agentId}/metrics \
  -H "X-Api-Key: sk-your-key" \
  -d '[{
    "name": "response_time",
    "value": 1.2,
    "unit": "seconds"
  }]'

API Endpoints Reference

Agents

  • POST /agents - Create agent
  • GET /agents - List with filters (category, status, model, team)
  • GET /agents/my-agents - List agents user belongs to
  • PATCH /agents/{id}/activate - Set status to Active
  • PATCH /agents/{id}/inactivate - Set status to Inactive
  • PUT /agents/{id}/team - Assign to specific team

Metrics (Public/API Key)

X-Api-Key Required

  • POST /agents/{id}/metrics - Push single metric
  • POST /agents/{id}/metrics/bulk - Push metric array
  • GET /agents/{id}/metrics?days={n} - Get history

Teams & Keys

  • POST /teams - Create new team
  • POST /teams/{id}/members - Manage team access
  • POST /agents/{id}/apikeys - Create ingestion key

Security & Compliance

Multi-Tenancy

The system enforces strict tenant isolation. Only users from authorized Entra ID tenants (configured via Auth:AllowedTenantIds) can access the platform.

Rate Limiting & Keys

Metrics ingestion is decoupled from user session tokens. Each X-Api-Key is bound to a specific agentId for maximum security and minimal attack surface.

Frequently Asked Questions

Do I have to use AI Hub for all projects?

No, it's voluntary but highly recommended for centralized visibility.

How does authentication work?

Users use Microsoft Entra ID (OIDC). Metrics collection uses X-Api-Key headers.
