Overview

AI Hub is a centralized management and observability platform for AI agents at 99x. It enables developers to register agents, monitor performance in real-time, and manage organizational access through a unified dashboard.

Architecture & Model

System Design

Designed as a high-performance system with clear layer separation:

  • Frontend: React/Next.js dashboard for management and visualization.
  • Backend: ASP.NET Core 8.0 API Gateway handling business logic and external integrations.
  • Persistence: PostgreSQL for relational data and identity management via Entra ID.

Core Data Model

  • Users: Managed via Office 365/Entra ID with multi-tenant support.
  • Agents: Each agent instance belongs to exactly one team.
  • Teams: Supports hierarchical structures (Parent/Child teams).
  • Roles: Access managed at the agent/team level (Admin vs. User).

Tech Stack & Infrastructure

Cloud Native: Hosted entirely on Azure using Infrastructure as Code (Bicep/Azure DevOps).

Quick Start Guide (Under 15 Mins)

1. Prerequisites & Auth

The platform uses Microsoft Entra ID. All direct API calls require a Bearer token.

  • Production Dashboard: staihub.z16.web.core.windows.net
  • Local Development: http://localhost:3000
  • API Base: http://localhost:5000 (Dev)

2. Register Your Agent

curl -X POST http://localhost:5000/agents \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Support Bot",
    "category": "CustomerService",
    "model": "GPT4"
  }'

3. Push Metrics (Single or Bulk)

Use your generated X-Api-Key to push data. You can send multiple metrics in one call for efficiency:

# Bulk Metrics Example
curl -X POST http://localhost:5000/agents/{id}/metrics/bulk \
  -H "X-Api-Key: sk-abcd1234" \
  -d '[
    {"name": "response_time", "value": 1.2, "unit": "seconds"},
    {"name": "tokens_used", "value": 150, "unit": "count"}
  ]'

API Reference

Agents & Teams

  • GET /agents/my-agents - Your agents
  • PATCH /agents/{id}/activate - Activate
  • PUT /agents/{id}/team - Assign Team
  • POST /teams/{id}/members - Add Members

Users & Keys

  • GET /users/me - Current Profile
  • GET /agents/{id}/apikeys - List Keys
  • DELETE /agents/{id}/apikeys/{keyId} - Revoke

Platform Integration

Universal Integration Model

AI Hub is designed to be platform-agnostic. You can integrate it with any system that can make HTTP requests (Python, Node.js, C#, or No-Code tools like Zapier/Make).

1. Internal Agents

Simply include the X-Api-Key in your agent's middleware to push logs, metrics, and costs after every execution.

2. CI/CD Pipelines

Integrate into GitHub Actions or Azure DevOps to update agent status or register new versions automatically upon deployment.

SDK & Language Examples

// Example: Node.js / TypeScript Integration
const pushMetric = async (name, value) => {
    await fetch(`https://api.aihub.99x.io/agents/${AGENT_ID}/metrics`, {
        method: 'POST',
        headers: {
            'X-Api-Key': process.env.AIHUB_API_KEY,
            'Content-Type': 'application/json'
        },
        body: JSON.stringify({ name, value, unit: 'seconds' })
    });
};
# Example: Python Integration
import requests

def report_usage(tokens):
    requests.post(
        f"https://api.aihub.99x.io/agents/{AGENT_ID}/metrics",
        headers={"X-Api-Key": AIHUB_API_KEY},
        json={"name": "token_usage", "value": tokens, "unit": "count"}
    )

Best Practices

Optimizing Your AI Implementation

  • Metric Frequency: Avoid flooding the API. Batch your metrics using the /bulk endpoint at the end of a request cycle or every 5 minutes.
  • Error Identification: Don't just report "Error". Use dimensions to specify if it was a RateLimit, ContextWindowOverflow, or SafetyGuardrail hit.
  • Security: Rotate your X-Api-Key every 90 days. Never hardcode keys; use environment variables or Azure Key Vault.
  • Versioning: When deploying a major model change (e.g., GPT-3.5 to GPT-4), register a new agent or use dimensions to track the version drift.

FAQ

How does authentication work?

Users use Entra ID. Metrics ingestion uses X-Api-Key headers bound to specific agents for security.

Is participation mandatory?

No, but highly recommended for centralized visibility and cost management.
